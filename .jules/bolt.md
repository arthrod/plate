## 2024-05-18 - [O(N*M) algorithmic complexity from Array.prototype.includes in selection filters]
**Learning:** In highly interactive components or event handlers (like drag selection in `packages/selection`), using `.includes()` inside loops or `.filter()` calls against potentially large arrays (like list of elements) causes an O(N*M) performance drop. In `packages/selection/src/internal/SelectionArea.ts`, filtering hundreds of selectable items against arrays of currently selected/stored items during `mousemove` created significant blocking on the main thread.
**Action:** When filtering or intersecting arrays of DOM nodes or objects, always initialize a `Set` for O(1) lookup before the loop (e.g., `const selectedSet = new Set(selected)` then `!selectedSet.has(node)`), reducing complexity to O(N).
