## 2025-03-09 - XSS Vulnerability in static media file nodes
**Vulnerability:** Found a Cross-Site Scripting (XSS) vulnerability in `apps/www/src/registry/ui/media-file-node-static.tsx` where user-provided `url` properties from `props.element` were injected directly into the `href` attribute of an anchor tag (`<a>`) without any sanitization.
**Learning:** This occurred because static representations of Slate/Plate elements are not immune to malicious payloads that are stored in structural JSON data. If a user inputs a `javascript:alert(1)` URI as a file download link, it would be rendered explicitly as a clickable vector.
**Prevention:** Always wrap user-provided URLs in HTML attributes (`href`, `src`) with the `sanitizeUrl` utility from `@platejs/utils` or `platejs` / `platejs/static` (e.g. `sanitizeUrl(url) || ''`). Crucially, never rely on specific allowlisted protocols since Plate relies on `data:` and `blob:` URIs, just ensure javascript/vbscript protocols are sanitized via this utility method.
